# 周会优化建议

## 问题现状
当前周会模式的弊端在于倾向于“流水账式汇报”，导致：
- **信息单向流动**，缺乏有价值的交互。
- **参会者关注度低**，只关心自己的部分，产生大量“陪跑”时间。
- **会议价值低**：如果周会只是为了“同步信息”，那么异步发一份文档效率可能更高。

## 核心目标
未来的周会应该从“汇报上周做了什么”，转变为“**关注当下遇到了什么困难，以及需要决策哪些事**”。

## 建议方案：异步 + 同步 结合

### 异步（会前）
- **时间**：各部门在每周五 18:00 前。
- **动作**：在飞书多维表格（或任何共享文档）上**全员更新项目进度**。
- **要求**：
    - 各部门**领导**负责确认信息准确性，并过滤掉无意义的琐事。
    - 进度状态需明确标记为：
        - 🟢 **绿色 (On Track)**: 进展顺利。
        - 🟡 **黄色 (At Risk)**: 存在风险，但可控/需要关注。
        - 🔴 **红色 (Blocked)**: 遇到阻碍，需要会议上**决策**。

### 同步（会上）

1.  **会前默读 (5分钟)**
    - 全员静默阅读共享表格中的内容。
    - 对任何需要讨论的项（即使是绿色项）进行**划线或评论提问**。

2.  **汇报与讨论 (核心环节)**
    - **汇报顺序**：严格按照 `🔴 -> 🟡 -> 🟢` 的优先级进行。
    - 🔴 **先过红色项 (重点讨论)**
        - **汇报人**: 清晰地提出问题 + 给出1-2个建议方案。
        - **决策人**: 听取讨论后立即拍板。
        - **结果**: 形成明确的 **Todo-list**，指定责任人和截止日期。
    - 🟡 **快过黄色项 (同步风险)**
        - **汇报人**: 仅同步风险点。
        - **主持人**: 确认风险责任人。
        - **结果**: 指定相关人员**会后小范围讨论**解决，不占用全体时间。
    - 🟢 **默认不过绿色项**
        - 除非有人在默读环节划线提问，否则会上不讲，节省所有人的时间。

## 为什么需要这么做

- 🟢 **绿色项（提供背景与留痕）**:
  1.  为黄、红项提供必要的上下文。
  2.  通过存档留痕，保留追溯项目进度的能力（主要对各部门领导和项目管理有价值）。
- 🟡 **黄色项（敏捷响应）**:
  1.  将“小病”扼杀在摇篮里，避免小风险拖成大事故。
  2.  通过快速同步，确保责任落实到人，且不占用全员大量时间。
- 🔴 **红色项（聚焦高价值产出）**:
  1.  这才是周会**最大的价值所在**。
  2.  通过节省掉讲绿色项的时间，将最宝贵的同步时间集中在需要被解决的重大问题上，并产出可执行的决策。
