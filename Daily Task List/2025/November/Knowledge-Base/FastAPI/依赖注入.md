# FastAPI 依赖注入 (Dependency Injection)

## 核心思想

只有路由函数（Web 层）负责“索要”依赖；服务函数（逻辑层）只负责“接收”和“使用”依赖。

## 一个比喻：餐厅

- **路由函数 (`handle_robot_message`)**: 就像餐厅的**服务员**。
- **服务函数 (`process_robot_messages`)**: 就像厨房的**厨师**。
- **依赖实例 (`DingTalkRobotFileDown`)**: 就像一把特殊的**厨刀**。
- **`Depends(...)`**: 就像服务员用来向后厨下单的**点菜机**。

### 错误的做法
厨师的菜谱上写着“请使用点菜机来切菜”。厨师拿到点菜机，完全不知道怎么用，于是报错了。
*(这就是在服务函数里出现 `Depends` 的情况)*

### 正确的做法
1.  **服务员** (`handle_robot_message`) 使用**点菜机** (`Depends`) 向后厨申请一把**厨刀**。
2.  后厨（FastAPI 框架）把厨刀实例交给了服务员。
3.  服务员把这把厨刀亲自送给**厨师** (`process_robot_messages(body, downloader=那把厨刀)`)。
4.  厨师拿到真正的厨刀，顺利完成切菜任务。

## 总结

请始终将 `Depends` 视为 Web 层（路由层）的专属工具，它的职责是从框架中获取资源。然后，将获取到的资源作为普通参数，清晰地传递给负责处理业务逻辑的服务层。

这样，您的代码职责就会非常分明，也不容易再犯同样的错误。
