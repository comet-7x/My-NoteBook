     ```json
     {
       "id": 1,
       "vector": [0.1, 0.2, 0.3],
       "name": "Item A",    // Not in schema
       "category": "books"  // Not in schema
     }
     ```

     启用动态字段功能后，Milvus 将其内部存储为：

     ```json
     {
       "id": 1,
       "vector": [0.1, 0.2, 0.3],
       // highlight-start
       "$meta": {
         "name": "Item A",
         "category": "books"
       }
       // highlight-end
     }
     ```

     

3. 什么是 ANN 和 KNN，以及 **ANN 在 Milvus 中** 的应用：

   - **ANN** 是 **Approximate Nearest Neighbor** 的缩写，中文是 **近似最近邻**

     - **核心思想：** 在一个大型数据集中，快速找到与给定查询点最相似（或最近）的 **$k$** 个数据点，但 **不保证** 找到的这 $k$ 个点是绝对精确的（即不是真正意义上的“最近”）。
     - **用途：** 主要用于 **大规模** 数据集的 **向量搜索**。由于精确搜索耗时太长，ANN 通过牺牲一小部分精度来换取巨大的 **查询速度提升**。

   - **KNN** 是 **$k$-Nearest Neighbor** 的缩写，中文是 **$k$ 最近邻**

     - **核心思想：** 在一个数据集中，找到与给定查询点 **距离最近的** **$k$** 个数据点，并且 **保证** 找到的就是精确最近的 $k$ 个点。

     - **用途：**

       1. **分类：** 根据这 $k$ 个最近邻居的类别，通过多数投票来决定查询点的类别。
       2. **回归：** 根据这 $k$ 个最近邻居的值，取平均或其他方式来预测查询点的值。
       3. **精确搜索：** 作为一种 **精确** 的最近邻搜索算法。

     - 在 Milvus 中，ANN 被用作为基本向量搜索方式，使用方式为根据搜索结果与查询向量的 **相似度得分** 从高到低排列搜索结果，其中 **度量类型** 是定义向量之间“相似”或“距离”的关键，它决定了 ANN 算法如何计算两个向量的近似性，以下是 Milvus 中提供的类型：

       | 度量类型  | 特征                         | 距离范围                    |
       | --------- | ---------------------------- | --------------------------- |
       | `L2`      | 值越小表示相似度越高。       | [0, ∞)                      |
       | `IP`      | 数值越大，表示相似度越高。   | [-1, 1]                     |
       | `COSINE`  | 数值越大，表示相  似度越高。 | [-1, 1]                     |
       | `JACCARD` | 值越小，表示相似度越高。     | [0, 1]                      |
       | `HAMMING` | 值越小，表示相似度越高。     | [0，dim(向量)] 批量向量搜索 |

       1. **L2（欧氏距离）：** 计算两个向量在多维空间中的直线距离。

          $$d(x, y) = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}$$

          - **特性：** 既考虑向量的 **方向**，也考虑向量的 **长度（模长）**。如果两个向量的方向和长度都相似，L2 距离就会很小。
          - **相似度判断：** **距离**，值越小越相似。
     
       2. **IP（内积）：** 计算两个向量对应元素的乘积之和。
     
          $$\text{IP}(x, y) = \sum_{i=1}^{n} x_i y_i$$
     
          - **特性：** 结果同时受向量的 **方向** 和 **长度** 影响。**向量越长，内积越大。**
     
          - **相似度判断：** **相似度**，值越大越相似。
     
       3. **COSINE（余弦相似度）：**
     
          $$\text{COSINE}(x, y) = \frac{\sum_{i=1}^{n} x_i y_i}{\sqrt{\sum_{i=1}^{n} x_i^2} \sqrt{\sum_{i=1}^{n} y_i^2}}$$
     
          - **特性：** **通过归一化消除了向量长度的影响**，只关注两个向量在空间中的 **夹角**。在文本、语音等领域，向量长度通常代表词频，而方向代表语义，因此余弦相似度更常用。
          - **相似度判断：** **相似度**，值越大越相似。
          - **注意：** 在 Milvus 内部，如果选择 `COSINE`，系统通常会在进行 IP 搜索之前，自动对所有向量进行 **L2 归一化**。因此，`COSINE` 搜索在底层可以视为对归一化向量执行的 **IP** 搜索。
     
       4. **JACCARD 与 HAMMING（针对二进制向量）**：这两种度量专门用于 **二进制向量**（元素只有 0 或 1），通常用于二值化后的特征数据。
     
          - **JACCARD 距离：** 适用于衡量两个集合的相似性。在二进制向量中，它关注两个向量中 **1** 的 **共同出现** 情况。
          - **HAMMING 距离：** 衡量两个向量的不同之处，即 **异或 (XOR)** 运算后结果中 **1** 的个数（不匹配的位数）。它常用于校验和纠错。

### 学习记录：

1. Linux软件的安装与卸载笔记：

   - 包管理器安装（推荐，自动解决依赖）：

     - Debian/Ubuntu 系统（apt）：

       ```bash
       # 1. 更新软件源（必须）
       sudo apt update
       
       # 2. 安装软件（推荐方式）
       sudo apt install 软件包名
       
       # 示例：安装VLC媒体播放器
       sudo apt update && sudo apt install vlc
       ```

   - 手动安装软件包：

     - 安装.deb 包（Debian/Ubuntu 系统）：

       ```bash
       # 方法1：使用dpkg（不自动解决依赖）
       sudo dpkg -i 软件包.deb
       
       # 方法2：使用apt安装（自动解决依赖，推荐）
       sudo apt install 软件包.deb
       ```

       

   - 确认软件的安装方式：

     ```bash
     # 1. 检查是否为 dpkg/apt 安装（最常用，如 FlClash）
     dpkg -l | grep 软件名  # 有输出则为 deb/apt 安装
     apt list --installed | grep 软件名  # 同上
     
     # 2. 检查是否为 snap 安装
     snap list | grep 软件名
     
     # 3. 检查是否为 flatpak 安装
     flatpak list | grep 软件名
     
     # 4. 检查是否为源码编译安装（无包管理记录，需手动定位）
     which 软件名  # 查看可执行文件路径
     locate 软件名  # 查看所有相关文件
     ```

     

   - 使用 **dpkg/apt** 安装软件的卸载（适配 Debian/Ubuntu 系）

     - 普通卸载（保留用户配置，可重装复用）：
       适合需要保留个人配置（如软件的自定义规则、账号信息）的场景，仅删除系统级程序文件。

       ```bash
       # 方法1：用 apt 卸载（自动处理依赖，推荐）
       sudo apt remove 软件名 -y
       
       # 方法2：用 dpkg 卸载（效果同 apt remove，无依赖自动清理）
       sudo dpkg -r 软件名
       ```

       

     - 彻底卸载（清除所有文件 + 配置，无残留）：

       适合需要完全清理、重新安装的场景（如软件配置错乱、缓存异常等）。若直接下载 `xxx.deb` 手动安装（未通过 apt 源），卸载命令一样

       ```bash
       # 步骤1：彻底删除软件包（purge 会删除系统配置文件）
       sudo apt purge 软件名 -y
       
       # 步骤2：清理无用依赖（卸载软件时残留的依赖库）
       sudo apt autoremove -y
       
       # 步骤3：清理 apt 缓存（可选，释放磁盘空间）
       sudo apt clean
       
       # 步骤4：删除用户级配置/缓存（包管理器不会自动清理，关键步骤）
       # 通用路径，根据软件调整
       rm -rf ~/.config/软件名
       rm -rf ~/.local/share/软件名
       rm -rf ~/.cache/软件名
       rm -rf ~/.软件名  # 部分软件会在用户根目录生成配置
       
       # 步骤5：验证是否卸载干净
       dpkg -l | grep 软件名  # 无输出则包已删除
       which 软件名  # 输出“无此文件”则可执行文件已删除
       locate 软件名 | grep -v "grep"  # 无核心文件输出则无残留
       ```
