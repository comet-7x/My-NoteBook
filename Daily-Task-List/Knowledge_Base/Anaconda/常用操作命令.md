
问题原因：
Anaconda 在安装时，会自动向 Windows 的 PowerShell/CMD 启动配置文件中添加**conda 初始化脚本**，并默认开启「自动激活 base 环境」的功能。因此每次打开终端，conda 会自动执行激活 base 环境的命令，终端前缀就会显示 `(base)`。

### 解决方法（分 2 种，推荐第一种）

#### 方法 1：通过 conda 命令永久关闭（推荐，无需改文件）

这是官方推荐的方式，直接禁用「自动激活 base 环境」的全局配置：

1. 打开 PowerShell/CMD（不管是否激活 base 都可以）；
2. 执行以下命令：
    ```bash
    conda config --set auto_activate_base false
    ```
    
3. 关闭当前终端，重新打开 PowerShell/CMD，即可看到 `(base)` 前缀消失，默认不再激活 base 环境。

**如果想恢复自动激活**，执行反向命令即可：

bash

运行

```bash
conda config --set auto_activate_base true
```

#### 方法 2：手动删除终端启动配置中的 conda 初始化脚本（适合进阶）

如果方法 1 无效，可能是 conda 初始化脚本仍在终端启动时执行，可手动移除：

##### 针对 CMD（命令提示符）

1. 打开「运行」（Win+R），输入 `regedit` 打开注册表编辑器；
2. 定位到路径：`HKEY_CURRENT_USER\Software\Microsoft\Command Processor`；
3. 找到名为 `AutoRun` 的项，如果其值包含 `conda init cmd.exe` 相关内容，右键删除该值（或清空值内容）；
4. 重启 CMD 即可。

##### 针对 PowerShell

1. 打开 PowerShell，执行命令找到 PowerShell 配置文件路径：
    ```bash
    echo $PROFILE
    ```
    
2. 按输出的路径打开该文件（如果文件不存在则忽略），例如路径可能是：`C:\Users\你的用户名\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1`；
3. 删除文件中所有以 `# >>> conda initialize >>>` 开头、`# <<< conda initialize <<<` 结尾的段落（这是 conda 自动添加的初始化脚本）；
4. 保存文件，重启 PowerShell 即可。

#### 临时关闭（仅当前终端生效）

如果只是想临时退出 base 环境（不修改全局配置），在终端中执行：

```bash
conda deactivate
```

执行后当前终端的 `(base)` 前缀会消失，但下次打开终端仍会自动激活。

### 验证是否生效

1. 关闭所有已打开的 PowerShell/CMD；
2. 重新打开一个新的终端窗口；
3. 观察终端前缀是否有 `(base)`：如果没有，说明配置生效。

---

### 总结

1. **核心原因**：Anaconda 安装时自动添加了初始化脚本，且默认开启「自动激活 base 环境」配置；
2. **推荐方案**：执行 `conda config --set auto_activate_base false` 永久关闭自动激活，简单且无风险；
3. **临时方案**：执行 `conda deactivate` 仅退出当前终端的 base 环境，不影响全局配置。
