
## `MagicMock()` 的使用

|源代码写法|Mock 赋值方式|解释|
|---|---|---|
|obj.attr（无括号）|mock.attr = "值"|模拟普通属性 / 变量|
|obj.method ()（有括号）|mock.method.return_value = "值"|模拟函数 / 方法的调用结果|
|func ()（顶层函数）|mock_func.return_value = "值"|模拟函数的调用结果|
|obj.a.b.c（链式）|mock.a.b.c = "值"|模拟深层属性|
|obj.method().attr|mock.method.return_value.attr = "值"|模拟方法返回值的属性|

## `return_value` vs `side_effect`

  在进行单元测试时，Mock 对象的这两个属性是最常用的，它们决定了当 Mock 对象被调用时“会发生什么”。
  1. 核心定义



  ┌───────┬─────────────────────────────────────────────────────┬─────────────────────────────────────────────────────────────────────────┐
  │ 特性  │ return_value (固定返回值)                           │ side_effect (副作用/动态行为)                                           │
  ├───────┼─────────────────────────────────────────────────────┼─────────────────────────────────────────────────────────────────────────┤
  │ 本质  │ 静态属性                                            │ 可执行对象（函数）、迭代器或异常                                        │
  │ 行为  │ 无论调用多少次，无论参数是什么，始终返回同一个预定… │ 每次调用时执行特定的逻辑。可以根据参数返回不同值、抛出异常或依次返回列… │
  │ **优… │ 较低                                                │ 较高。如果同时设置了两者，side_effect 会覆盖 return_value。
  │
  └───────┴─────────────────────────────────────────────────────┴─────────────────────────────────────────────────────────────────────────┘

  ---

  2. 详细对比

  A. return_value：简单、确定
  适用于不需要逻辑判断、只需要一个结果来让代码继续运行的场景。


   * 内存表现：返回的是该对象的引用。如果返回的是可变对象（如列表、字典），在一个地方修改了它，下次调用返回的依然是那个被修改后的对象。
   * 示例：


   1     mock_func = MagicMock()
   2     mock_func.return_value = 200
   3     
   4     mock_func() # 返回 200
   5     mock_func(a=1) # 依然返回 200

  B. side_effect：复杂、灵活
  适用于需要模拟真实逻辑、测试异常处理或模拟流式数据的场景。


  常见用法：
   1. 绑定函数（动态逻辑）：根据传入的参数动态决定返回值。
   1     def dynamic_resp(val):
   2         return val + 1
   3     mock_func.side_effect = dynamic_resp
   2. 模拟工厂（创建新实例）：每次调用返回一个全新的对象（如 ServerSentEvent），避免对象状态污染。
   3. 抛出异常：测试代码的 try...except 块。
   1     mock_func.side_effect = Exception("连接超时")
   4. 迭代返回：传入一个列表，每次调用按顺序返回列表中的下一个值。
   1     mock_func.side_effect = [1, 2, 3]
   2     mock_func() # 返回 1
   3     mock_func() # 返回 2

  ---

  3. 场景选择建议


  什么时候用 return_value？
   * 模拟配置项（如获取 settings.API_KEY）。
   * 模拟简单的状态返回（如布尔值 True/False）。
   * 不需要关注参数，只需要函数“给个面子”回个值即可。


  什么时候用 side_effect？
   * 模拟 SSE/流式数据：需要每次返回不同的 Data 块或全新的 Event 实例。
   * 测试错误处理：需要函数在特定情况下崩掉（抛出错误）。
   * 参数相关性：需要 mock(id=1) 返回 A，而 mock(id=2) 返回 B。
   * 有状态的调用：模拟一个计数器或数据库游标（每次调用结果不同）。

  ---

  4. 特别注意：优先级
   1 mock_obj = MagicMock(return_value="A")
   2 mock_obj.side_effect = lambda *args: "B"
   3
   4 print(mock_obj()) # 输出 "B"，因为 side_effect 优先级更高


  5. 针对你项目的提示
  在你的 knowledge_chat 测试中：
   * 使用 side_effect = mock_sse_event 是为了确保每次调用 send_xxx_node 时，产生的是全新的、独立的 `ServerSentEvent`
     对象，这符合流式传输每条消息都是独立实体的真实物理逻辑，能有效防止测试脚本中的对象引用混淆。
