# Linux软件安装与卸载指南

## 1. 安装方式概览

### 包管理器安装（推荐）
这是最常规和安全的方式，能自动处理依赖关系。

- **Debian/Ubuntu 系统 (apt):**
  ```bash
  # 1. 更新软件源（必须）
  sudo apt update
  
  # 2. 安装软件
  sudo apt install 软件包名
  
  # 示例：安装VLC媒体播放器
  sudo apt update && sudo apt install vlc
  ```

### 手动安装软件包
适用于没有被软件源收录的程序。

- **安装 `.deb` 包 (Debian/Ubuntu):**
  ```bash
  # 方法1：使用 dpkg（不自动解决依赖）
  sudo dpkg -i 软件包.deb
  
  # 方法2：使用 apt 安装（自动解决依赖，推荐）
  sudo apt install ./软件包.deb # 注意路径
  ```

## 2. 如何确认软件的安装方式
在卸载前，先确认软件是通过哪种方式安装的。

```bash
# 1. 检查是否为 dpkg/apt 安装（最常用）
dpkg -l | grep 软件名
# 或
apt list --installed | grep 软件名

# 2. 检查是否为 snap 安装
snap list | grep 软件名

# 3. 检查是否为 flatpak 安装
flatpak list | grep 软件名

# 4. 检查是否为源码编译安装（无包管理记录）
which 软件名  # 查看可执行文件路径
locate 软件名 # 查看所有相关文件
```

## 3. 卸载软件 (以 dpkg/apt 为例)

### 普通卸载 (保留配置)
适合想保留个人数据（如规则、账号信息）的场景。

```bash
# 方法1：用 apt 卸载（自动处理依赖，推荐）
sudo apt remove 软件名 -y

# 方法2：用 dpkg 卸载（无依赖自动清理）
sudo dpkg -r 软件名
```

### 彻底卸载 (清除所有文件)
适合需要完全清理、重装的场景（如配置错乱、缓存异常）。

```bash
# 步骤1：彻底删除软件包（purge 会删除系统级配置文件）
sudo apt purge 软件名 -y

# 步骤2：清理无用依赖（卸载软件时残留的库）
sudo apt autoremove -y

# 步骤3：清理 apt 缓存（可选，释放磁盘空间）
sudo apt clean

# 步骤4：删除用户级配置/缓存（关键步骤，包管理器不管这里）
# 注意：请根据实际软件名称替换 "软件名"
rm -rf ~/.config/软件名
rm -rf ~/.local/share/软件名
rm -rf ~/.cache/软件名
rm -rf ~/."软件名" # 部分老旧软件会在用户根目录生成配置

# 步骤5：验证是否卸载干净
dpkg -l | grep 软件名              # 应无输出
which 软件名                       # 应提示 "not found"
locate 软件名 | grep -v "grep"   # 应无核心文件输出
```
