**智能分页显示**：如果内容不满一屏，直接显示；如果超过一屏，再进入跳转界面。

**在终端执行：**
```bash
git config --global core.pager "less -F -X"
```
- **参数解释：**
    
    - `-F` (Quit if one screen)：如果内容不满一屏，自动退出跳转模式，直接显示在屏幕上。
        
    - `-X` (No init)：防止 `less` 在退出时清空屏幕内容。
        
- **效果：** 以后运行 `git branch -v` 时，如果分支不多，它会直接显示在终端；如果分支非常多，它才会跳转让你翻页。

**验证是否成功：** 你可以运行以下命令查看配置是否已经写入：
```bash
git config --global core.pager
```
如果返回 `less -F -X`，说明已经生效。


**检查 `.gitconfig` 文件（可选）：** 你也可以直接查看 Git 的配置文件内容：
```bash
cat ~/.gitconfig
```

你会发现里面多了一行：

```ini, TOML
[core]
    pager = less -F -X
```

当你执行 `git config --global core.pager "less -F -X"` 时，你实际上是在修改 Git 交付数据的方式。

### 1. 核心知识点：Pager（分页器）

在 Unix/Linux 系统中，**Pager** 是一个专门用于处理“长文本”的工具。

- **为什么要它？** 想象一下，如果你有 10,000 行日志，如果不经过分页器直接打印，你的终端会飞速滚动，你根本看不清内容，而且会消耗大量内存来缓存这些滚动历史。
    
- **常见的 Pager：** `more`（老古董）和 `less`（现代标准）。Git 默认使用 `less`。
    

#### 为什么以前会跳转？

默认情况下，Git 只要看到输出内容，就会无条件地把数据丢给 `less`。而 `less` 的默认行为是：**开辟一个全屏缓冲区，等待用户操作（按 Q 退出）。**

---

### 2. 参数解析：为什么这几个字母能解决问题？

当你设置 `less -F -X` 时，你是在给这个“中转站”下达特定的指令：

- **`-F` (Quit if one screen)：**
    
    - **逻辑：** 告诉 `less` 先计算一下内容的行数。
        
    - **功能：** 如果内容很少，**不满一屏**，`less` 会在把内容显示出来后**立即自动退出**。这就是为什么你的 `git branch -v` 不再需要按 `q` 的原因。
        
- **`-X` (No init)：**
    
    - **逻辑：** 禁用终端的“备用屏幕”切换功能。
        
    - **功能：** 默认情况下，`less` 退出时会清空屏幕，让你回到运行命令前的样子。加上 `-X` 后，`less` 退出时内容会**保留在终端屏幕上**。这符合我们查看分支名并继续输入下一条命令的操作习惯。

### 3. 你需要了解的相关扩展知识

为了让你更深入地掌握这类操作，以下是三个建议关注的知识点：

#### A. Git 配置层级

你运行的 `git config --global` 是修改了你的用户级配置。Git 的配置有三个层级：

1. **System (--system):** 对电脑上所有用户生效。
    
2. **Global (--global):** 对当前登录用户的所有项目生效（最常用）。
    
3. **Local (--local):** 仅对当前所在的 Git 仓库生效。
    

#### B. 环境变量的优先级

如果你在 `.zshrc` 中写了 `export PAGER=cat`，它会覆盖 `git config` 里的设置吗？

- **优先级顺序：** `GIT_PAGER` 环境变量 > `core.pager` 配置 > `PAGER` 环境变量。
    
- 了解这一点能帮你排查为什么有时候配置了却不生效。
    

#### C. 如何恢复默认

如果你哪天想换回原来的样子，或者觉得这个设置不好用，可以运行：

Bash

```
git config --global --unset core.pager
```

这会删除该配置，让 Git 回归初始状态。
