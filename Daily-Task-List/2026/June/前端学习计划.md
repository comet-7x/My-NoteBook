# 前端学习计划:HTML + CSS + JS + TS(后端工程师快速通道)

> 适用对象:已熟练 Python 后端、懂 async/作用域/类型注解、毕设用过 Vue3 的人。 核心原则:**只学和已知不一样的部分**。跳过"什么是变量"这类内容,把时间压在 CSS 和实战上。 预期:全职专注 2~4 周到"能独立写中等复杂度前端"。CSS 的打磨会持续几个月,属正常。

------

## 阶段总览

| 周      | 主题                  | 对你的难度           | 目标产出                             |
| ------- | --------------------- | -------------------- | ------------------------------------ |

## 第 1 周:HTML + CSS

### Day 1 — HTML(半天搞定,别停留)

- [ ] 过一遍语义化标签:`header / nav / main / section / article / aside / footer`
- [ ] 表单:`form / input / label / select / textarea / button`,以及 `name`/`value`/`type` 的作用
- [ ] 可访问性基础:`alt`、`label for`、语义标签为什么重要(给屏幕阅读器/SEO)
- [ ] 理解"HTML 没有逻辑,它只是结构标记"这一点,然后就别再花时间了

### Day 2~3 — CSS 核心三块(死磕)

- [ ] **盒模型**:`box-sizing: border-box`、margin/padding/border、外边距合并
- [ ] **Flexbox**:玩通关 [Flexbox Froggy](https://flexboxfroggy.com/)
- [ ] **Grid**:玩通关 [Grid Garden](https://cssgridgarden.com/)
- [ ] 选择器与优先级(specificity)、层叠规则、继承
- [ ] 单位:`px / rem / em / %  / vw / vh`,何时用哪个

### Day 4 — 定位与响应式

- [ ] `position`:static / relative / absolute / fixed / sticky
- [ ] 媒体查询 `@media`,移动优先(mobile-first)写法
- [ ] 常见布局模式:居中、两栏、卡片网格、粘性页头
- [ ] 简单过渡 `transition`(为后面聊天界面的动效铺垫)

### Day 5 — CSS 小项目(必须动手)

- [ ] 不用任何框架,手写一个响应式个人页(头像 + 简介 + 项目卡片网格)
- [ ] 要求:窄屏单列、宽屏多列,完全靠 Flexbox/Grid 实现
- [ ] 卡在哪就搜 MDN,记录下你"以为会但其实不会"的点

> ✅ 本周验收:能脱离教程,凭盒模型 + Flexbox/Grid 独立排出一个不崩的响应式页面。

------

## 第 2 周:JavaScript(语言 + 运行环境)

> 主力教材:[javascript.info](https://javascript.info/)。对有编程基础的人最高效。 跳过基础语法章节,直奔下面这些"和 Python 不一样的地方"。

### Day 6 — 语言差异速通

- [ ] `let / const`、模板字符串、解构、展开运算符、可选链 `?.`
- [ ] 数组方法:`map / filter / reduce / find / forEach`(和 Python 推导式对照着记)
- [ ] 对象、`JSON.parse/stringify`
- [ ] 真值/假值、`==` vs `===`(这是坑,务必搞清)

### Day 7 — JS 的"特有难点"(重点)

- [ ] `this` 绑定规则(普通函数 vs 箭头函数)—— Python 没有,必看

- [ ] 闭包(你懂,快速确认)、原型链与 `class`

- [ ] 事件循环:宏任务/微任务、为什么 

  ```
  setTimeout(0)
  ```

   不是立刻执行

  - 你后端踩过 `asyncio.sleep(0)` 的流式问题,这里概念同源,会很快通

### Day 8 — DOM 操作

- [ ] `querySelector / querySelectorAll`、`createElement`、`append/remove`
- [ ] 修改内容与属性:`textContent`、`innerHTML`(注意 XSS)、`classList`、`dataset`
- [ ] 这是 JS 真正"新"的部分——后端没有 DOM,需要练

### Day 9 — 事件与异步

- [ ] 事件模型:`addEventListener`、事件冒泡/捕获、事件委托
- [ ] `Promise`、`async/await`(概念你已会,确认 JS 写法)
- [ ] **`fetch` API**:发请求、处理响应、错误处理
- [ ] ESM 模块:`import / export`(和 Python import 对照)

### Day 10 — JS 小项目(动手)

- [ ] 用原生 JS 做一个调你后端某个普通接口的页面:点击按钮 → fetch → 渲染结果列表
- [ ] 要求:有 loading 态、有错误态、用事件委托处理列表项点击
- [ ] 全程不准用框架,目的是让 DOM/事件/异步形成肌肉记忆

> ✅ 本周验收:能用原生 JS 完成"取数据 → 操作 DOM → 处理交互与异步"全流程。

## 第 3 周:TypeScript + 工具链

> TS 对你来说是"加强版 Python typing",1~2 天足够。剩下时间给工具链。

### Day 11 — TS 语言

- [ ] 基础类型、`interface` vs `type`、联合/交叉类型
- [ ] 泛型(对照 Python `TypeVar`)
- [ ] 工具类型:`Partial / Pick / Omit / Record / Readonly`
- [ ] 结构化类型(structural typing)——和 Python 鸭子类型同源,理解这点就通了
- [ ] `tsconfig.json` 关键项:`strict`、`target`、`module`、`paths`

### Day 12 — 现代工具链

- [ ] Node 基础:`node` 运行、`package.json`、`npm/pnpm` 装包
- [ ] **Vite**:`npm create vite@latest`,选 vanilla-ts 模板,跑通 dev/build
- [ ] 搞清 Vite 干了什么(开发服务器 + 打包),**不要去学 Webpack 配置**
- [ ] ESLint + Prettier 基础配置(团队协作必备)

### Day 13 — TS + DOM 实战

- [ ] 把第 10 天那个原生 JS 项目用 Vite + TS 重写一遍
- [ ] 给接口返回数据定义 `interface`,体会类型带来的安全感
- [ ] 处理 `document.querySelector` 的返回类型(`HTMLElement | null`)与断言

### Day 14~15 — 缓冲 / 补课

- [ ] 回填前两周没吃透的点(大概率是 CSS 布局或 `this`/事件循环)
- [ ] 读一点真实开源前端代码,适应工程结构

> ✅ 本周验收:能用 Vite + TS 起工程,给数据流定义类型,工具链不再是黑盒。

## 第 4 周:杀手级实战项目 —— 流式聊天前端

> 别做 to-do list。给你自己的 Agent(COMET-AGENT 那套 SSE + Redis Pub/Sub)写前端。 它会同时逼你用上 HTML 结构 + CSS 布局 + JS 异步流 + TS 类型,而且完成后是**能用的真东西**。

### Day 16 — 搭骨架

- [ ] Vite + TS 工程初始化
- [ ] 页面结构:消息列表区 + 输入框 + 发送按钮
- [ ] 定义类型:`Message { role: 'user' | 'assistant'; content: string }`

### Day 17 — 静态 UI

- [ ] 用 Flexbox 排消息气泡(左右区分 user/assistant)
- [ ] 消息区可滚动、新消息自动滚到底
- [ ] 输入框与发送按钮的布局与禁用态

### Day 18~19 — 接 SSE 流式(核心)

- [ ] 用 `fetch` + `ReadableStream` 或 `EventSource` 消费你后端的 SSE 接口
- [ ] 把流式 token 逐块追加渲染到当前 assistant 气泡(打字机效果)
- [ ] 处理:流开始(建空气泡)、流中(追加)、流结束、报错中断
- [ ] 这一段直接对应你后端的流式输出,前后端打通会很有成就感

### Day 20 — 打磨

- [ ] loading / 发送中 / 错误态的 UI
- [ ] 简单动效(气泡淡入、滚动平滑)
- [ ] 代码块/换行的基本渲染(若 Agent 返回 markdown,可先做最简版)
- [ ] 提交到 GitHub(你的 `comet-7x`),写个像样的 README

> ✅ 项目验收:浏览器里能和你自己的 Agent 流式对话,token 实时蹦出来。
